# Requirements: Guardião Firmware — Production Hardening

**Defined:** 2026-04-11
**Core Value:** Firmware ESP8266 confiável e maintível que coleta temperatura e sincroniza com Firebase sem travamentos ou reboots inesperados.

## v1 Requirements

Requisitos para o milestone de produção (endurecimento). Cada um mapeia para fases do roadmap.

### Reliability (Confiabilidade)

- [ ] **REL-01**: Watchdog ISR não chama Serial nem nenhuma função não ISR-safe (apenas sinaliza flag)
- [ ] **REL-02**: Dispositivo tenta WiFi.reconnect() antes de recorrer ao ESP.reset()
- [ ] **REL-03**: Auth no Firebase em setup() tem contagem máxima de tentativas com exibição de erro no OLED antes de reiniciar
- [ ] **REL-04**: Falhas no envio ao Firebase usam exponential backoff (não reset imediato)
- [ ] **REL-05**: Fetch de configurações ocorre em intervalo separado e mais longo do que o envio de dados

### Code Quality (Qualidade de Código)

- [ ] **CQ-01**: drawOled.h é dividido em módulos focados (rendering, buttons, alarm, buzzer)
- [ ] **CQ-02**: Valores sentinela de temperatura (-127.0, 888.0) são substituídos por flag bool sensorError
- [ ] **CQ-03**: Concatenação de String em hot-path usa snprintf em buffers char[] onde o comprimento é previsível

### Observability (Observabilidade)

- [ ] **OBS-01**: Falhas de boot exibem tela de erro nomeada no OLED antes de reiniciar
- [ ] **OBS-02**: Desconexão WiFi exibe status de reconexão no OLED

### Testing (Testes)

- [x] **TEST-01**: Ambiente de teste nativo PlatformIO configurado com framework Unity
- [x] **TEST-02**: Testes unitários para calculateStatusBitmap(), checkThresholdAlert(), encode/decode EEPROM

## v2 Requirements

Adiados para release futuro. Monitorados mas fora do roadmap atual.

### Resilience

- **RES-01**: Suporte a MQTT como protocolo alternativo de comunicação (quando Firebase está indisponível)
- **RES-02**: Cache local de leituras em caso de perda total de conectividade (SPIFFS/LittleFS)

### Observability Avançada

- **OBS-03**: Dashboard serial com métricas de uptime, contagem de reboots e falhas de envio

### Hardware Futuro

- **HW-01**: Suporte a versão B (ESP32-S/S3) com mesma base de lógica
- **HW-02**: Integração LoRa para versão C

## Out of Scope

Explicitamente excluído. Documentado para prevenir scope creep.

| Feature                                    | Motivo                                                            |
| ------------------------------------------ | ----------------------------------------------------------------- |
| OTA automático sem confirmação             | Risco em produção — dispositivo pode ficar inacessível            |
| Múltiplos sensores DS18B20 por dispositivo | Não é necessário no hardware v1 (PCB existente)                   |
| Interface web embarcada                    | WiFiManager já cobre configuração inicial; painel é no app mobile |
| Criptografia do payload Firebase           | Auth token JWT já provê segurança de transporte suficiente        |
