<objective>
Substituir String concatenation por snprintf+char[] nos hot-paths de Firebase para reduzir heap fragmentation no ESP8266.

Purpose: Cada String("...") + String(x) aloca heap dinâmico → fragmentação → eventual crash em MCU com 80KB RAM. Hot-paths executam a cada 30s.
Output: main.cpp e EEPROMFunction.h usam char buf[128] + snprintf em vez de String+.
</objective>

  <name>Task 1: snprintf para hot-paths em main.cpp e EEPROMFunction.h</name>
  <files>src/main.cpp, src/EEPROMFunction.h</files>
  <action>
Para cada hot-path abaixo, substituir `String path = "..." + String(x) + "..."` por `char buf[N]; snprintf(buf, sizeof(buf), "...", x);` e passar `buf` onde antes passava `path.c_str()`.

**Hot-paths (chamados a cada 30s):**

1. **sendDataToFireBase() — datalogger paths** (main.cpp ~linhas 319-320):

   ```cpp
   // ANTES:
   json.set(("/datalogger/" + String(timestamp) + dataloggerTempPath).c_str(), ...)
   json.set(("/datalogger/" + String(timestamp) + dataloggerTimestampPath).c_str(), ...)
   // DEPOIS:
   char pathBuf[96];
   snprintf(pathBuf, sizeof(pathBuf), "/datalogger/%lu%s", (unsigned long)timestamp, dataloggerTempPath);
   json.set(pathBuf, ...);
   snprintf(pathBuf, sizeof(pathBuf), "/datalogger/%lu%s", (unsigned long)timestamp, dataloggerTimestampPath);
   json.set(pathBuf, ...);
   ```

   NOTA: dataloggerTempPath e dataloggerTimestampPath são `const char*` ou `String`. Se forem String, usar `.c_str()`. Verificar tipos no código.

2. **drainOnePendingReading()** (EEPROMFunction.h ~linha 109):
   ```cpp
   // ANTES:
   String drainPath = "/datalogger/" + String(pendingQueue[0].timestamp) + ...
   // DEPOIS:
   char drainPath[96];
   snprintf(drainPath, sizeof(drainPath), "/datalogger/%lu/temperature", (unsigned long)pendingQueue[0].timestamp);
   ```
   Ajustar chamadas subsequentes para usar `drainPath` diretamente (já é `const char*`).

**Warm-paths (chamados no boot/heartbeat — menos críticos mas fáceis):**

3. **sendBootMessage()** (main.cpp ~linha 732):

   ```cpp
   // ANTES:
   String bootPath = "/UsersData/" + uid + "/devices/" + DEVICE_LDID + "/boot";
   // DEPOIS:
   char bootPath[128];
   snprintf(bootPath, sizeof(bootPath), "/UsersData/%s/devices/%s/boot", uid.c_str(), DEVICE_LDID);
   ```

   NOTA: DEVICE_LDID é `char[]` (de config.h). uid é `String` (de Firebase auth).

4. **sendHeartbeat()** (main.cpp ~linha 750):

   ```cpp
   // ANTES:
   String heartbeatPath = "/UsersData/" + uid + "/devices/" + DEVICE_LDID + "/heartbeat";
   // DEPOIS:
   char heartbeatPath[128];
   snprintf(heartbeatPath, sizeof(heartbeatPath), "/UsersData/%s/devices/%s/heartbeat", uid.c_str(), DEVICE_LDID);
   ```

5. **setup() — databasePath** (main.cpp ~linha 581):

   ```cpp
   // ANTES:
   databasePath = "/UsersData/" + uid + "/devices/" + DEVICE_LDID;
   // DEPOIS — databasePath é String global, trocar para char[]:
   ```

   CUIDADO: `databasePath` é usado em muitos locais. Se for `String` global, a conversão exige mudar todos os usos. Se databasePath é usado com `.c_str()` em Firebase calls, converter para `char databasePath[128]` e ajustar definição em config.h. Se os usos de databasePath são extensos demais, MANTER como String neste ponto (setup roda uma vez, baixo impacto) e adicionar comentário `// TODO: convert to char[] when refactoring config.h`.

6. **generateOrLoadLDID()** (main.cpp ~linha 691):
   ```cpp
   // ANTES:
   DEVICE_LDID = "ldid-" + String(DEVICE_UUID) + "-001";
   // DEPOIS:
   snprintf(DEVICE_LDID, sizeof(DEVICE_LDID), "ldid-%s-001", DEVICE_UUID);
   ```
   NOTA: Verificar se DEVICE_LDID é `char[]` ou `String`. Se já é `char[]`, snprintf funciona diretamente. Se é String, manter String neste caso (setup path).

Para cada substituição, a regra é:

- 96 bytes para paths curtos (/datalogger/XXXXXXXXXX/temperature)
- 128 bytes para paths longos (/UsersData/UID/devices/LDID/boot)
- sizeof(buf) SEMPRE como segundo arg do snprintf (previne overflow)
