Localizar o bloco de display em loop() (após 02-01 ter sido executado). O bloco atual é:

```cpp
if (updateAlarm()) {
  // Alarm active — display managed by alarm screen
} else if (displayMode == DMODE_NORMAL) {
  ui.update();
}
// DMODE_MODAL: display managed by handleButtons (hold progress)
```

Substituir o `else if (displayMode == DMODE_NORMAL)` branch por (per D-18):

```cpp
if (updateAlarm()) {
  // Alarm active — display managed by alarm screen
} else if (displayMode == DMODE_NORMAL) {
  if (WiFi.status() == WL_CONNECTED) {
    ui.update();
  } else {
    drawWifiReconnecting();
  }
}
// DMODE_MODAL: display managed by handleButtons (hold progress)
```

Sem delay, sem yields extras, sem flags de estado — retorno automático ao ui.update() quando WiFi reconectar (per D-20).

NOTA: overlayStatusBar() já mostra icon_wifi_off quando WiFi cai. drawWifiReconnecting() substitui os ui frames normais — os dois são complementares e corretos em paralelo.
